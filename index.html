Product Development Workflow
— Implementation Blueprint

Built on Deuce's framework. This document covers both the workflow design and the technical automation that makes it run.

Core Principle

COO gets full visibility and planning control. Developer keeps execution flexibility. Automation handles the glue — task routing, board updates, project creation.

COO Gets

  • Real-time visibility into progress, blockers, completions
  • Planning control — ranked backlog, capacity budgets, gate reviews
  • Scope protection — PRD approval before large builds start

Developer Gets

  • Automated task capture — Slack requests become Asana tasks
  • Minimal board maintenance — automations keep projects in sync
  • Clear priorities — ranked backlog, no ambiguity on what's next

Intake & Routing

Size-Based Routing KEPT

Different sizes get different process. XS/S skip scoping entirely.

flowchart TD
    TASK["New Task Enters Pipeline"] --> SIZE{"Size?"}
    SIZE -->|"XS / S"| SKIP["Auto-Skip to Backlog Top"]
    SIZE -->|"M"| SCOPE["Define → Scope → Backlog"]
    SIZE -->|"L / XL"| CHECK{"New Project?"}
    CHECK -->|"Yes"| CREATE["Auto-Create Feature Project\nfrom Template"]
    CHECK -->|"No"| SCOPE_LXL["Full PRD → Scope Review\n→ COO Approves"]
    CREATE --> SCOPE_LXL
    SKIP --> PULL["Developer Pulls into Active Work"]
    SCOPE --> PULL
    SCOPE_LXL --> PULL
SizeTypePathApproval
XS / SBug, config changeIntake → Auto-skip to Backlog topNone
MEnhancement, automationIntake → Define → Scope → BacklogCOO reviews
L / XLMulti-feature systemIntake → Define → Scope → Feature ProjectCOO + stakeholders

TECHNICAL XS/S Auto-Skip

Asana native rule: IF "Size" = XS or S → move to Backlog section. No Make.com needed.

Technical — Intake

Slack Intake Command AUTOMATION

Team captures work from any Slack conversation. AI reads thread context and generates a structured Asana task.

flowchart TD
    A["/task triggered in Slack"] --> B["Modal: Project, Type, Link, Screenshot"]
    B --> C["Backend captures form + full thread"]
    C --> D["Claude AI generates title, description, size"]
    D --> E["Asana task created in Pipeline"]
    E --> F["Confirmation posted to Slack"]

Form Fields

  • Project Association — all projects + "New Project"
  • Type — New Project | Bug & Quick Fix | Feature
  • Link — optional
  • Screenshot — optional upload

AI Backend

  • Captures message + full thread context
  • Claude API generates title + description
  • Auto-suggests size (XS/S/M/L/XL)
  • Creates Asana task with all fields

Technical — Project Creation

New Project Auto-Setup AUTOMATION

When a task is designated as a new project, the system creates a Feature Project from template with everything pre-configured.

flowchart TD
    A["Type = 'New Project'\nOR Association = 'New Project'"] --> B["Make.com triggers"]
    B --> C["Create Project from Template\n4 phases + milestones"]
    C --> D["Active Work field pre-attached"]
    C --> E["Copy PRD from Pipeline task"]
    C --> F["Add to Portfolio"]
    D --> G["Update Project Association dropdown"]
    E --> G
    F --> G
    G --> H["Slack notification with link"]

Trigger

  • Type = "New Project"
  • OR Project Association = "New Project"

Template Creates

  • 4 phases: Scope → MVP → Refinement → Completion
  • 3 milestones: Scope Review, MVP Review, Completion Review
  • Active Work field + multi-home rule pre-attached

Critical: Dropdown Update

After creating the project, the automation adds a new option to "Project Association" dropdown via Asana API — future /task commands immediately show the new project.
